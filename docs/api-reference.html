Overview

The OFKMS API provides REST access to the OpenFrame Knowledge Management System. It runs a 5-phase cascading search pipeline (embedding search, domain knowledge, OFCode web docs, parser, LLM fallback) and returns structured answers with source attribution and confidence scoring.

Base URL

http://your-host:8000

Content Type

All request bodies must be JSON. Set Content-Type: application/json.

Response Format

All responses are JSON. Streaming endpoints use Server-Sent Events (SSE).

Authentication

Authenticate requests by including your API key in the X-API-Key header. API keys are stored as SHA-256 hashes in PostgreSQL and linked to user accounts.

The following endpoints do not require authentication: /v1/health, /v1/admin/login, /docs, /redoc, /.
HeaderValue
X-API-KeyYour API key (created via POST /v1/admin/keys)

If the API key is missing or invalid, the server returns:

{
  "error": "Invalid or missing API key"
}

HTTP Status: 401 Unauthorized

Bootstrap: On first startup, a default admin user and API key are auto-created and logged to the console. Use these credentials to get started.

POST/v1/query

Send a question about OpenFrame products. The system runs the full search pipeline and returns a structured answer.

POST/v1/query/stream

Same as /v1/query but returns Server-Sent Events (SSE) with real-time phase progress.

The request body is identical to /v1/query.

Event Types

EventDescription
phasePhase completion notification (phase number, name, time)
answerFinal answer with confidence, language, intent, product
donePipeline complete with total_time_ms
errorError occurred during processing

Event Stream Format

event: phase
data: {"phase": 0, "name": "query_analysis", "status": "complete", "time_ms": 5}

event: phase
data: {"phase": 1, "name": "embedding_search", "status": "complete", "time_ms": 1200}

event: phase
data: {"phase": 2, "name": "domain_knowledge", "status": "complete", "time_ms": 800}

event: phase
data: {"phase": 3, "name": "ofcode_web", "status": "complete", "time_ms": 300}

event: answer
data: {"answer": "## BMS and MFS Comparison\n...", "confidence": 0.85, "language": "ja", "intent": "comparison", "product": "openframe_osc_7"}

event: done
data: {"total_time_ms": 3200}
Use the browser's EventSource API or any SSE-compatible HTTP client to consume the stream.

User Management

Manage user accounts. A default admin user is auto-created on first startup with credentials logged to the console.

POST/v1/admin/login

Verify user credentials. No API key required.

ParameterTypeRequiredDescription
usernamestringRequiredUsername
passwordstringRequiredPassword

Response

messagestring
user_idint
usernamestring
rolestringadmin | user
curl -X POST http://localhost:8000/v1/admin/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "SecureAdm1nP@ss2024!"}'

POST/v1/admin/users

Create a new user. Admin only.

ParameterTypeRequiredDescription
usernamestringRequiredUnique username (1-100 chars)
passwordstringRequiredPassword (min 8 chars)
rolestringOptionaladmin or user (default: user)

GET/v1/admin/users

List all users with active key counts. Admin only.

PATCH/v1/admin/users/{user_id}

Update user details (username, password, role, is_active). Admin only.

DELETE/v1/admin/users/{user_id}

Deactivate a user and revoke all their API keys. Admin only.

Key Management

API keys are per-user and stored as SHA-256 hashes. Keys are shown only once at creation time. All key endpoints require a valid X-API-Key header.

POST/v1/admin/keys

Create a new API key for the authenticated user.

ParameterTypeRequiredDescription
namestringOptionalLabel for the key (max 200 chars)

Response

api_keystringThe generated key (ofkms-...) — shown only once
key_idintDatabase ID (use for revocation)
namestring | null
messagestring
curl -X POST http://localhost:8000/v1/admin/keys \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-new-key"}'

Quick Start Guide:
1. Start the server: uvicorn api.server:app --port 8000
2. Check console logs for the auto-generated admin credentials and API key
3. Verify login: curl -X POST http://localhost:8000/v1/admin/login -H "Content-Type: application/json" -d '{"username":"admin","password":"SecureAdm1nP@ss2024!"}'
4. Use the API key: curl -H "X-API-Key: ofkms-..." http://localhost:8000/v1/products
5. Create more keys: curl -X POST http://localhost:8000/v1/admin/keys -H "X-API-Key: ofkms-..." -H "Content-Type: application/json" -d '{"name":"my-key"}'

Error Codes

The API uses standard HTTP status codes.

CodeMeaningDescription
200OKRequest succeeded
400Bad RequestInvalid request body (missing query, bad language code)
401UnauthorizedMissing or invalid API key
422Validation ErrorRequest body fails Pydantic validation
500Server ErrorInternal pipeline error
503Service UnavailableAll infrastructure services are down

Error Response Format

{
  "error": "Invalid or missing API key",
  "detail": null
}

Examples

1. Basic Query (Japanese)

curl -X POST http://localhost:8000/v1/query \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"query": "TJESとは何ですか？"}'
import httpx

resp = httpx.post(
    "http://localhost:8000/v1/query",
    headers={"X-API-Key": "your-api-key"},
    json={"query": "TJESとは何ですか？"},
)
data = resp.json()
print(data["answer"])
const resp = await fetch("http://localhost:8000/v1/query", {
  method: "POST",
  headers: {
    "X-API-Key": "your-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ query: "TJESとは何ですか？" }),
});
const data = await resp.json();
console.log(data.answer);

3. Streaming (SSE)

curl -N -X POST http://localhost:8000/v1/query/stream \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"query": "Protrieveのオプションテーブルについて教えてください"}'
import httpx

with httpx.stream(
    "POST",
    "http://localhost:8000/v1/query/stream",
    headers={"X-API-Key": "your-api-key"},
    json={"query": "Protrieveのオプションテーブルについて教えてください"},
) as resp:
    for line in resp.iter_lines():
        if line.startswith("data: "):
            print(line[6:])
const eventSource = new EventSource(
  "http://localhost:8000/v1/query/stream",
  // Note: EventSource only supports GET. For POST, use fetch + ReadableStream:
);

// Using fetch for POST SSE:
const resp = await fetch("http://localhost:8000/v1/query/stream", {
  method: "POST",
  headers: {
    "X-API-Key": "your-api-key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    query: "Protrieveのオプションテーブルについて教えてください",
  }),
});
const reader = resp.body.getReader();
const decoder = new TextDecoder();
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  console.log(decoder.decode(value));
}
